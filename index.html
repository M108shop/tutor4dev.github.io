Getting Started - Git and GitHub

Chapter 1 - Introduction

Git

Git เป็น Distributed Version Control System รองรับการทำงานทั้งแบบ Online และ Offline

GitHub

GitHub เป็น Project Hosting Service สำหรับจัดเก็บ Repository ที่ใช้ Git เป็น VCS รองรับฟังชั่นให้นักพัฒนาที่ไม่ใช่เจ้าของ Repository สามารถเข้ามามีส่วนร่วมในการร่วมพัฒนาโปรเจค โปรเจคที่มีชื่อเสียงแทบทั้งหมดเช่น jQuery, Bootstrap, Laravel, pallarax, etc ต่างก็ใช้ GitHub ในการจัดเก็บ Repository แทบทั้งสิ้น

Gist

Gist เป็น Service ภายใต้ GitHub ให้บริการแชร์ Code Snippet สนับสนุน Version Control System และมีฟังชั่นรองรับให้นักพัฒนาที่ไม่ใช่เจ้าของ Code Snippest สามารถมีส่วนร่วมในการแก้ไขเพิ่มเติมเพื่อให้ Code Snippest ดังกล่าวมีความสมบูรณ์ยิ่งขึ้น

วัตถุประสงค์ในการใช้ VCS

  • เพื่อพัฒนาโปรเจคอย่างมีระบบ
  • VCS ช่วยในการ Track การพัฒนาโปรเจค
  • VCS ช่วยในการ Track Who, What, When, Where, Why
  • นำ History จากการ Track มาใช้ในการบริหารจัดการโปรเจค
  • สำรองข้อมูล

กิจวัตรประจำวันของนักพัฒนา

  • สร้างโปรเจค
  • แก้ไข
  • จัดเก็บ
  • แก้ไข & จัดเก็บ
  • แก้ไข & จัดเก็บ
  • รีวิว, แก้บั๊ก
  • แก้ไข & จัดเก็บ

จุดเด่นของ Git

  • ฟรี
  • รองรับทุกแพลตฟอร์ม
  • รองรับทุกภาษาในการพัฒนาโปรแกรม
  • เร็ว, เล็ก, ไม่กินทรัพยากรของระบบ
  • ทำงานได้ทั้งแบบ Offline/Online (Distrubuted)

git push -u origin master

คำสั่งพื้นฐานอื่นๆ ของ Git

คำสั่ง git tag

ใช้ในการกำหนดเลขเวอร์ชั่นเพื่ออ้างอิง Commit

#สร้าง tag ให้กับ Commit ปัจจุบัน
git tag 0.9.9
#สร้าง tag ให้กับ Commit ที่ต้องการ
git tag @hash

คำสั่ง git diff

ใช้ในการเปรียบเทียบความแตกต่างระหว่าง Commit

#เปรียบเทียบปัจจบันกับการ Commit ล่าสุด
git diff

#เปรียบเทียบเฉพาะไฟล์ที่กำหนด
git diff filename

#เปรียบเทียบระหว่าง Commit
git diff @hash1 @hash2

#เปรียบเทียบระหว่าง Commit เฉพาะไฟล์ที่กำหนด
git diff @hash1 @hash2 filename

เลื่อน HEAD Pointer ไปยัง Commit ที่ต้องการ

#เลื่อน HEAD ไปยัง Commit ที่ต้องการ
git checkout @hash
#เลื่อน HEAD กลับมายัง Commit ล่าสุด
git checkout master

#เลื่อน HEAD ไปยัง tag ที่ต้องการ
git checkout tag#
#เลื่อน HEAD กลับมายัง Commit ล่าสุด
git checkout master

ลบไฟล์ออกจากโปรเจค

#ลบไฟล์
git rm filename
#Commit
git commit -m "Delete filename"

เปลี่ยนชื่อ/ย้ายไฟล์ในโปรเจค

#เปลี่ยนชื่อไฟล์
git mv filename new_filename
#Commit
git commit -m "Rename filename to new_filename"

.gitignore

#สร้างไฟล์ .gitignore
touch .gitignore
#เพิ่มชื่อไไฟล์หรือแฟ้มที่ไม่ต้องการให้ Git ทำการ Track

#Stage ไฟล์
git add .gitignore
#Commit
git commit -m "Add .gitignore"

ยกเลิกก่อนการ Commit (Unstaged)

เป็นคำสั่งอันตราย เพราะ Git จะยกเลิกการแก้ไขไฟล์ทั้งหมด

#ยกเลิกทั้งหมด
git checkout .

#ยกเลิกเฉพาะไฟล์ที่กำหนด
git checkout -- filename

ยกเลิกหลังจากการ Commit

#ยกเลิก Commit โดยเก็บข้อมูล
git reset --mixed HEAD^

#ยกเลิก Commit โดยทิ้งข้อมูล (คำสั่งอันตราย)
git reset --hard HEAD^

Chapter 4 - แตกสาขาการพัฒนา (Branch)

Branch คือ Separate Workspace สำหรับการพัฒนาโปรเจค

วัตถุประสงค์ของการใช้งาน Branch ?

  • แก้ไขบั๊กของโปรเจค
  • ทดลองคุณสมบัติใหม่ๆ ให้กับโปรเจค
  • แยกเวอร์ชั่น dev ออกจาก stable เวอร์ชั่น
  • พัฒนาโปรเจคในแบบทีม

คำสั่ง Branch พื้นฐาน

แสดง Branch ทั้งหมด

#แสดงเฉพาะ Local
git branch

#แสดงทั้ง Local และ Remote
git branch -v

สร้าง Branch ใหม่

git branch new_branch_name

สลับใช้งาน Branch

#สลับใช้งาน Branch ใหม่
git checkout new_branch_name

#สลับกลับมาใช้งาน Branch หลัก
git checkout master

ทำความเข้าใจเรื่อง Pointer

  1. HEAD
  2. master
  3. branch_name
  4. origin/master
ภาพตัวอย่างโปรเจคที่มี 1 Branch (master)

ภาพตัวอย่างโปรเจคที่มี 2 Branch (master, testing)

เปรียบความแตกต่างระหว่าง Branch

#เปรียบเทียบความแตกต่างทั้งหมด
git diff master branch_name

#เปรียบเทียบความแตกต่างเฉพาะไฟล์ที่กำหนด
git diff master branch_name filename

ลบ Branch

git branch -d branch_name

การ Merge

คำสั่งในการ Merge

#เปลี่ยนมาใช้งาน master
git checkout master

#Merge branch_name มารวมกับ master
git merge branch_name

#ลบ branch_name
git branch -d branch_name

Chapter 1 - bower

bower เป็น Package Manager ที่ใช้สำหรับติดตั้ง Front-End Library ต่างๆ เช่น

  • jQuery
  • Twitter Bootstrap
  • Angular.js
  • parallax
  • font-awesome
  • etc.

bower Requirements

  • npm
  • git

ติดตั้ง bower

ติดตั้ง bower

npm install -g bower

ทดสอบ bower

#เวอร์ชั่น
bower --version

#แสดงคำสั่งทั้งหมด
bower --help

ใช้งาน bower

สร้างไฟล์ bower.json

bower init

ค้นหา Package

bower search what_to_search

ติดตั้ง Package

bower install package_name

#ติดตั้ง Bootstrap
bower install bootstrap --save-dev

#ติดตั้ง Font Awesome
bower install font-awesome --save-dev

#ติดตั้ง jQuery โดยการกำหนดเวอร์ชั่น
bower install jquery#1.1 --save-dev
bower install jquery#2 --save-dev

แสดง Package ที่ถูกติดตั้งแล้ว

bower list

Update Package เป็นเวอร์ชั่นล่าสุด

bower update

ถอนการติดตั้ง

bower uninstall package_name --save-dev
